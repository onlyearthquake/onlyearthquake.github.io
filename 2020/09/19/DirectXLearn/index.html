向量

DirectX中，变量向量使用XMVECTOR，将会借助SIMD对四维向量进行同时运算（而不是一个个分别计算它们的值)

类中的向量成员应使用XMFLOATn（n为向量的维数）进行存取

也存在着转换XMVECTOR和XMFLOAT的方法

XMVECTOR XM_CALLONV XMLoadFloat2(const XMFLOAT2 * psource)

void XM_CALLONV XMStoreFloatn(XMFLOATn *pDestination , FMVECTOR V)

为了使代码不受编译器的影响，将用四种不同的类型传递XMVECTOR参数

  • 前三个XMVECTOR为 FXMVEOTR
  • 第四个为GXMVECTOR，第五、六个为HXMVECTOR
  • 其余的为CXMVECTOR

此规则仅仅适用于输入XMVECTOR参数（将根据不同的硬件环境储存在SIMD或栈里），输出将正常储存在栈里

D3D的初始化

预备知识

纹理格式：储存2D图像数据，例如：

DXGI_FORMAT_R32G32B32_FLOAT 每个元素为32浮点数

DXGI_FORMAT_R16G16B16A16_UNORM四个16位分量，均能被映射到0~1间

DXGI_FORMAT_R16G16B16A16_SNORM均能被映射到-1~1之间

DXGI_FORMAT_R8G8B8A8_UINT八位，映射到0~255间

深度缓冲便是一种纹理，需要用明确的数据格式来创建

  • DXGI_FORMAT_D32_FLOAT_S8X24_UINT
    • 缓冲区+模板缓冲区+对齐区
  • DXGI_FORMAT_D32_FLOAT
  • DXGI_FORMAT_D23_UNORM_S8_UNIT
  • DXGI_FORMAT_D16_UNORM

资源和描述符

  • CBV constant buffer view
  • SRV shader resource view
  • UAV unordered target view
  • RTV render target view
  • DSV depth/stencil view

多重采样（抗锯齿）

SSAA：4倍分辨率的虚拟屏幕，4个像素一组解析

MSAA：同样是4X像素，但并不全部进行计算，而是计算像素中心颜色，基于可视性和覆盖性分享给子像素
