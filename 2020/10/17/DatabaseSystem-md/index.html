SQL基础

基本类型

  • char(n) 恒定长度的字符串

  • varchar(n) 长度可变的字符串，最大长度为n

  • int 整数类型；smallint 小整数类型

  • numeric(p,d) 定点数，共p位数字，小数点后d位

  • real, double precision 浮点数与双精度浮点数

  • float(n) 精度至少为n的浮点数

example:

create table

  • primary key(A1 , …) 主码声明
  • foreign key(A1,..) 外码声明 references（set ）表示来自哪个关系
  • not null 不允许空值

insert

insert:插入元素值；delete删除元素

drop table删除关系表；alter改变属性

查询的基本结构

单关系查询

select P from R where condition...

可使用distinct强行删除重复，也可以使用all来显式指明不去除重复

select子句还可以带有+、-、*、/运算符的基本表达式

where 筛选出在from子句的关系中满足特定谓词的元组，也可同时使用and、or、not等逻辑连词

多关系查询

from后如果跟着多个关系，会自动生成笛卡尔积

有属性名称相同时，应指明属于哪个关系

实际上，sql有自然连接(两个关系中都存在的属性相同时产生的元组)的连接词natural join

附加的基本运算

更改名称 as,可以更改属性的名称，也可以给关系一个别名·

字符串运算

like匹配模式：

  • %：匹配任意子串
  • _:匹配任意一个字符

如：

‘Intro%’任意以Intro开头的字符串

‘%Comp%’包含Comp子串的字符串

‘ _’匹配只含三个字符的字符串

为使模式中能包含特殊模式的字符（如%、_等），使用escape来定义转义字符

like ‘ab\%cd%’ escape ‘\‘

where子句谓词

between…and…

集合运算

union 并；intersect 交；except 差

空值

Sql中，涉及空集的运算结果均为unkown

  • and: unkown的结果均为unkown
  • or: true or unkown -> true;false or unknown -> unkown
  • not :unkown
  • is null

聚集

聚集函数

  • avg平均值；sum总和
  • min最小值；max最大值
  • count计数

以一个值的集合为输入、返回多个值的函数

分组聚集：

group by:在group by子句中所有属性取值上相同的元组被分在同一个组中

分组聚集中，出现在select中的非聚集属性一定出现在了group by中，否则是错误的

having子句，针对group by形成的分组作用

同样，任何出现在having而不聚集的属性，一定要存在于group by 中，否则错误

空值的聚集：聚集函数会自动忽略空值（count（*）除外）
