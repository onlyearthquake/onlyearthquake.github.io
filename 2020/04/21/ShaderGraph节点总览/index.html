Artistic

语义

Unity支持的语义

从模型数据到语义

  • POSITION 模型顶点位置
  • NORMAL 顶点法线 通常是float3
  • TANGENT 顶点切线，通常是float4
  • TEXCOORDn，如TEXCOORD0 表示纹理坐标
  • COLOR 顶点颜色 通常是fixed4或float4类型 ——SV_TARGET

从frag到vert

  • SV_POSITION 裁剪空间的顶点坐标
  • COLOR0 第一组顶点颜色
  • TEXCOORDn纹理坐标

SV_Target

Debug

  • 颜色的值永远在0和1之间

同时处理屏幕图像和纹理时，不同渲染平台可能会影响（如抗锯齿）

应提前判断是否是DirectX平台

#if UNITY_UV_STARTS_AT_TOP

if (_MainTex_TexelSize.y < 0) uv.y = 1- uv.y

#end if

Shader的语法差异

  • DirectX语法更严格，初始化必须提供所有参数，不能省略
  • DirextX11不允许使用tex2D函数（UV在vert阶段无法得到，如使用，应用tex2Dlod代替，加上#pragma target 3.0是shader model3.0的特性）

书写Shader注意点

  • 慎用分支控制（尽量在上流使用）
  • 不要除以0

渲染路径

前向渲染

处理光照方式：逐顶点、逐像素、SH

光照模式设为Important意味着尽量使用逐像素光照

  • 最亮的平行光按逐像素处理
  • NotImportant，按逐顶点或SH处理
  • 上述得到的逐像素光源数量小于QualitySetting中设置的数量，会有更多光源变为逐像素光源

BasePass与AdditionPass

  • Base默认支持阴影，Additional不支持，使用#pragma multi_compile_fwadd_fullshadows代替#pragma multi_compile_fwadd
  • 环境光和自发光在Base中计算
  • Additional开启并设置混合模式
  • 一个Base仅会执行一次，但Addition会根据逐像素光线的数量执行多次

可以通过函数访问前8个光源的信息

untiy_LightColor unity_LightPosition unity_LightAtten untiy_SpotDirection

延迟渲染

一个Pass处理G缓冲，一个Pass处理光照

  • RT0 ： 漫反射
  • RT1：高光；RT2：法线
  • RT3：自发光+lightMap+探针
  • 深度与模板缓冲

  • Point

    lightCoord = mul(untiy_WorldToLight,float4(i.vertex,1.0))

    atten = tex(_LightTexture0,dot(lightCoord,lightCoord).rr).UNITY_ATTEN_CHANNEL

  • Spot

    atten = (lightCoord.z > 0) * tex2D(_LightTexture0,lightCoord.xy / lightCoord.w + 0.5).w*tex2D(_LightTextureB0 , dot(lightCoord,lightCoord).rr).UNITY_ATTEN_CHANEL
