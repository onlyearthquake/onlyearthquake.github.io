浮点数的表示

“浮”点：小数点移动，但移动的浮点数不便于进行统一的计算

IEEAStandard：S + W + Frag（即符号位+幂数+尾数）

舍入：舍到最接近的偶数

浮点数不具有结合律

（1e5 + 1e-5) + 3.14 1e5 + (1e-5 + 3.14)

编译

Different

汇编：

  • 不同类型的整数：1,2,4,8
  • 浮点数的特殊处理：不同的寄存器
  • 每条指令的作用有限

r代表64位 e代表32位

无e：16 2字节

2字节中h代表高位 l代表低位
现代寄存器均可随意使用，其名字大多来自于早期遗产（除了s）

movq

movq source,dest

  • Immediate常数
  • register 寄存器地址
  • memory 内存地址

实际上只有5种可能组合（Immediate不能作为dest，不允许从一个memory读到另一个中）

（R）：使用寄存器

D（R）：寄存器地址+位移

D(Rb,Ri,S) Mem[Reg[Rb] + S * Ri + D]

函数参数：rdi,rsi

rip:控制点地址 rsp：执行代码地址

符号位：

CF ZF OF SF

设置：

cmpq b,a 置位为a -b，并不储存

testq b,a 置位为a&b 设置ZF和SF

获取：

set系列，将寄存器的值设置为0/1,根据相关的符号位

movqzbl 将一个字节移过来，左边全置为0

条件分支

jX:跳转指令

条件移动：同时执行两条语句，最终再决定取值

编译器优化：可以优化掉无用的初次测试代码

Switch

建jump表：从最低到最高的值建表，指向代码段

先判断是否在范围内，不在就default，缺省值通过查表到default

through:默认确实会直接传过去，代码是这样的

break:就是ret

特殊优化：赋值优化（比如在switch前w = 1，后又有w = 2，实际上w = 1的代码不会统一执行，而是存放在需要的代码段中）

Procudure

Stack

->用于管理调用与返回的状态。分配本地数据

-》递减栈指针实现栈

push pop

Calling Convention

Passing Control

call:递减栈指针，存入当前地址，跳转地址（change rip）到新函数中

Passing Data

前六个参数的寄存器：rdi rsi rda rcx r8 r9,超过的参数放到栈上

return val:rax

Managin local data

栈帧：分块的栈内存

一般情况下代码空间固定，只需要一个指针就可以实现释放和分配

但部分可变大小的特殊情况意味着需要两个指针

Caller Saved

想要不变，先存储起来

r10,r11

Callee Saved

被调用的函数改变寄存器的值时，先存到栈里，用完恢复

rbx,r12,r13.r14
