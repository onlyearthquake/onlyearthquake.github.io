    DNS

    主机名到实际IP的映射，一般分为三类：根服务器，TLD服务器，权威服务器

    type

    • A 主机名->ip
    • cname 别名 ->真名
    • ns 域名 -> 权威服务器主机名
    • mx 解析名 ->邮件服务器名

    DNS访问会产生缓存

    P2P

    ​ 所有对等方->torrent，当要下载的时候，与torrent中的所有对等方尝试建立并行的TCP连接（对等方的IP由追踪器发送），从每个连接中获取单个的小文件块（实际上不需要对等方完全具有这个文件，单个块也可以，甚至可以进行文件块的交换），依照最稀缺优先。

    CDN

    内容分发服务器，分为推高速缓存和拉高速缓存（一个定时上传内容，一个缓存未命中拉取）

    运输层

    多路分解与多路复用：分解：从运输层发给上层应用数据，复用：从应用数据分解到运输层报文段

    UDP：二元组，源端口和目的端口

    TCP：四元组，加上源IP和目的IP

    可靠连接协议

    完全可靠的链路：不需要什么状态，正常发送接收即可

    可能传错的链路：

    • 增加检验位

    发送端：等待调用和等待回复两个状态，回复NAK重传

    接收端：等待调用一个状态，正确就发送ACK错误发送NAK

    • 考虑到检验位出错的可能

    为什么需要分组的标志位：不知道传过来的是错误的重传还是新的分组。

    发送端：四个状态等待->等待0回应->等待1调用->等待1回应

    接收端：等待0->等待1

    也可以进一步简化为只有ACK，对另一个分组的ACK相当于NAK

    • 考虑到丢包

    增加计时器 到时间视为NAK，重传数据

    流水线传输

    N步传输：多次发送，如果接收方没有接收到有序的数据全部（当前未确认的数据）重传。

    选择重传：可以不有序，但接收方需要维护数据缓存。

    合理配置窗口和序号大小的关系（窗口最大也得比序号小2，不然容易出现严重错误）

    TCP

    报文段：源端口、目的端口；序号、确认序号；标志位、长度；接收窗口、检验和；紧急数据指针；选项；

    可靠连接的重传工作：

    • 超时间隔加倍（考虑到网络拥堵，每次超时的时间加倍）
    • 快速重传（接收到乱序的数据，发送亢余ACK，多次亢余ACK直接导致快速重传）
    • 选择确认（GBN的基础上，如果分组的ACK已到达，不会重传）

    连接与关闭的过程

    • 连接请求
    • SYN
    • SYN确认
    • 发送FIN
    • 接收两次ACK
    • 关闭

    拥塞避免：

    • 慢启动
      • 从1开始，2的n次方指数增长，知道大于等于上次发生拥塞的MSS一半
    • 拥塞避免
      • 线性增加 当遇到丢包或者亢余ACK时，进入快速恢复状态
    • 快速恢复
      • 直接减半

    IP

    IP路由选择

    路由表信息：

    • 目的IP地址：完整的主机地址/网络地址
    • 下一跳的地址：下一个中继/目的的IP地址
    • 标志。标志是网络地址还是主机地址+
    • 指定网络接口

    子网寻址

    主机号->子网号+主机号

    分出子网的好处是隐藏了内部组织的细节+缩小了路由表的规模

    子网掩码代表的其实是一种子网划分方式

    环回地址：从127.0.0.1到127.255.255.255 就是本机，广播消息会发一份到环回地址上

    变长子网

    SIMPLIEX选项 禁止接口不能收到本机传送的数据（会通过环回接口？）

    ARP

    地址解析：

    什么地址？ IP地址；为什么需要解析？ IP地址和链路层需要一个映射

    实际上是一个动态的解析

    ARP请求:目的地址 + 源地址 + 帧类型 硬件类型+协议类型+硬件长度+协议长度+操作类型+发送端以太网+发送IP地址+目的以太网+目的IP

    大致流程

    who-has -> machine in the net reply -> get the disk address

    RARP

    用于无磁盘的主机IP地址的获取

    无盘系统 本身没有系统，而是在局域服务器上

    ICMP

    类型 + 代码 检验和

    主机之间交换的报文，不用指定目的端口号

    发生错误会发送ICMP差错报文，但差错报文不会再次引起另一个差错报文。

    此外：广播地址、非主机地址都不会产生差错报文。

    ICMP地址掩码：类似RARP，无盘系统获取地址掩码

    ICMP时间戳：用来调整主机时间与系统时间的差值（difference - 1/2 RTT)

    ICMP端口不可达差错：

    UDP从特定端口到特定端口

    几个程序

    ping

    判断是否可达；-R选项获取报文路径

    路由路径信息会附加在报文的末尾

    Traceroute

    依次发送刚好TTL寿命到尽头的报文（目的主机可以发送一个不可达端口的UDP报文）

    两种ICMP不可达报文：TTL归0，或者正在分组

    然后让路由/主机发送不可达报文来确认路径

    选路

    路由表：决定报文被分发到哪里

    • 主机地址
    • 网络地址
    • 默认项

    都不符合就会发送不可达的差错报文

    ICMP重定向报文[类型 检验和 发送重定向的路由 原来的路由 更改之后的路由]

    主机在引导之后会广播或者多播路由请求报文，路由器返回路由通告报文

    配置的静态路由文件往往只是默认的 路由还是会定期广播更新路由表的

    广播和多播

    广播

    • 本地环回255.255.255.255
    • A类地址的广播
    • 子网的广播

    多播

    • 向多个目的地址传送数据
    • 客户端的请求

    254.XXXXXXXXX

    有一些知名多播

    • 224.0.0.1 子网所有系统组
    • 224.0.0.2 所有路由器组
    • 224.0.1.1网络时间协议

    由网卡进行过滤
